# Summary Format

Keep restore output compact and actionable.

Required order:

1. Context Header
2. Lifecycle Chain
3. Current State
4. Requirement Coverage
5. Blockers
6. Repo Status
7. External / Source Context
8. Learning Context
9. Inconsistencies
10. Recommended Next Action

Rules:

- Lead with resolved slug/version and confidence.
- Use tables for chain and repo state.
- Keep blockers concrete.
- Name exact paths.
- End with one recommended action and reason.
- Do not include implementation plans or broad advice.

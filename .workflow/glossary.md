# Glossary

| Term | Meaning |
|---|---|
| Artifact | Durable Markdown record for one lifecycle phase. |
| Requirement Manifest | Brief section containing `R`, `RI`, `A`, and `Q` IDs. |
| `R` ID | Explicit user requirement. |
| `RI` ID | Implicit requirement derived from repo, domain, source, verification, release, or safety context. |
| `A` ID | Assumption that must stay visible. |
| `Q` ID | Open question that may block a phase. |
| Blocker | Missing decision, access, evidence, or gate that prevents progress. |
| Waiver | Explicit acceptance of known residual risk. |
| Source of truth | Configured external or repository-local authority for requirements, docs, release notes, or task state. |
| Generated output | Derived files that must be checked against source or regeneration rules. |
| Ship | Lifecycle phase that decides release, handoff, source, rollback, and waiver readiness. |
| Reflect | Lifecycle phase that records outcome, follow-ups, and learning candidates. |

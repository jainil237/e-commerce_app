# Learning Capture

Reflect proposes learning candidates; it does not curate them by default.

Candidate types:

- Candidate learning: durable operating rule for future agents.
- Candidate skill update: change to a skill contract or reference.
- Candidate template update: change to an artifact template.
- Candidate config/schema update: change to machine-readable configuration.
- Candidate validator update: check that should become automated later.

Rules:

- Propose at most three high-signal candidates.
- Tag every candidate `propose-only`.
- Cite source artifact or evidence.
- Prefer specific, repeatable rules over generic advice.
- Do not edit `.workflow/learnings/curated.md` unless the user explicitly requests curation.
- Do not treat one-off project details as generic learnings unless the pattern is reusable.

# Unrelated Changes Policy

Unrelated changes belong to the user unless proven otherwise.

Build must:

- identify unrelated changes before edits
- avoid staging unrelated files
- avoid formatting or rewriting unrelated files
- avoid reverting user changes
- record unrelated overlapping files as blockers when they affect planned work

If an unrelated change is in a file Build must edit:

1. Read the file carefully.
2. Preserve the unrelated change.
3. Make the smallest compatible edit.
4. Record the overlap in the task artifact.
5. Ask the user only when the overlap makes safe implementation impossible.

If unrelated changes are outside scope, ignore them except for status reporting.
